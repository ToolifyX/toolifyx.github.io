# 🚀 PROMPT: Update Project Config for GitHub Pages (ToolifyX)

You are working on a React (Vite-based) project called **ToolifyX**.

## 🎯 Task

Update the project configuration to support **GitHub Pages deployment** and ensure all routes, assets, and build outputs work correctly in a static hosting environment.

---

## ⚙️ Required Changes

### 1. Vite Base Path Configuration

Update `vite.config.ts`:

* Set correct `base` path for GitHub Pages:

  * Format: `"/<main>/"`
* Ensure assets are correctly resolved in production build

---

### 2. Package.json Scripts

Add or update deployment scripts:

* `build`: production build
* `predeploy`: runs build before deploy
* `deploy`: uses `gh-pages` to publish `dist` folder

Ensure `gh-pages` is properly configured.

---

### 3. React Router Fix (IMPORTANT if used)

If the project uses React Router:

* Replace `BrowserRouter` with `HashRouter`
  OR
* Configure fallback for SPA routing on GitHub Pages

Ensure routes do not break on refresh.

---

### 4. Static Asset Handling

Ensure:

* All images and assets use correct relative paths
* No hardcoded absolute server paths (e.g. `/uploads`)
* File uploads use browser APIs only (`FileReader`, `URL.createObjectURL`)

---

### 5. Build Output

Ensure build output is:

* `dist/` folder (Vite default)
* Fully static and deployable

---

### 6. GitHub Pages Compatibility Checklist

Make sure project:

* Works without backend
* Loads correctly on refresh
* Supports direct URL access
* Has correct asset resolution

---

## 🌐 Goal Result

After changes:

* Project can be deployed using `npm run deploy`
* App runs correctly on:
  `https://github.com/ToolifyX/toolifyx.github.io`
* No broken routes or missing assets

---

## 🧠 Important Constraint

Do NOT introduce backend dependencies.
Keep everything fully static and GitHub Pages compatible.
