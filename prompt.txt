Build a **professional Image Resizer tool** inspired by iLoveIMG, fully integrated into ToolifyX, with a split layout where the **LEFT panel shows multiple image previews generated from the right-side configuration**.
🎯 Goal

Create a browser-based Image Resizer that supports:

* Percentage-based resizing
* Exact dimension resizing (width/height)
* Batch-style preview system
* Real-time preview generation based on selected settings
🖥 Layout (UPDATED — IMPORTANT)

#LEFT (70–80%) — Preview Workspace (PRIMARY)

This area is NOT a single image preview anymore.

It must display a **LIST OF IMAGE PREVIEWS** generated dynamically from the current configuration in the right panel.

Each item includes:

* Original image thumbnail
* Resized preview result
* Resolution info (original → new size)
* Visual comparison (before/after optional)
* Loading real-time updates when config changes

👉 Behavior:

* Every change in right panel immediately updates ALL previews
* Supports multiple uploaded images (batch mode)
* Scrollable grid layout (responsive)
#RIGHT (20–30%) — Controls Panel (SECONDARY)

Compact configuration panel controlling all previews in LEFT.
⚙️ Resize Modes

#1. Percentage Resize Mode

* Slider or numeric input (%)
* Example: 25%, 50%, 75%, 100%, 150%
* Applies to ALL images in preview list
* Live update across all items
#2. Dimension Resize Mode

* Inputs:

  * Width (px)
  * Height (px)

#Options:

* ☑ Maintain aspect ratio

  * Lock ratio when modifying width or height
* ☑ Do not enlarge if smaller

  * Prevent upscaling beyond original image size
🧠 Resize Behavior Rules (GLOBAL)

* All settings apply to **entire image list**
* LEFT panel always reflects current configuration state
* Changes in RIGHT panel instantly re-render all previews
* No manual refresh required
🧩 Functional Requirements

#Image Input

* Drag & drop multiple images
* File picker upload (multi-select supported)
* Supported formats: JPG, PNG, WEBP, HEIC
* Immediately populate LEFT preview list
#Preview System (LEFT PANEL CORE)

Each image item must show:

* Original image
* Resized result (live computed)
* Before/after comparison toggle (optional)
* File size estimate (optional)
* Dimensions (original → output)

Layout style:

* Grid (desktop)
* Single column (mobile)
* Smooth lazy rendering for performance
#Resize Engine

* Canvas-based processing
* Real-time batch processing
* Optimized for multiple images (10–50+ files)
* No UI lag when updating config
#Export System

* Export:

  * Single image download
  * Batch download (ZIP)
* Formats:

  * PNG (default)
  * JPG (quality slider 0–100)
🎨 UI/UX Requirements

* LEFT panel = dynamic preview gallery (primary workspace)
* RIGHT panel = control center (global settings)
* Modern SaaS UI (iLoveIMG + Figma-like batch tool style)
* Smooth real-time updates
* Clean grid layout, no clutter
* Dark mode support
🧱 Technical Requirements

* Fully client-side (no backend)
* Canvas API / Web image processing
* Efficient batch processing (worker recommended)
* Modular structure:

  * `ResizeGallery.tsx` (LEFT preview grid)
  * `ResizeControls.tsx` (RIGHT panel)
  * `useBatchResizeEngine.ts`
  * `resizeUtils.ts`
⭐ UX PRINCIPLE (CRITICAL)

* RIGHT panel controls everything
* LEFT panel is a **live batch preview system**
* Every config change = instant update of all images
* Must feel like a professional batch image processing tool (iLoveIMG level or better)
